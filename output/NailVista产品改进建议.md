# NailVista 产品功能模块改进建议

> 基于对当前项目实现的全面分析，从产品思维出发，提出能解决用户痛点、有创新性的功能设计。

---

## 一、当前产品能力盘点

| 模块 | 当前状态 | 痛点 |
|------|---------|------|
| AI 试戴 | 图片上传 + MediaPipe 合成 / Bailian API 生成 | 一次性操作，用完即走，无留存 |
| 款式浏览 | 分类筛选 + 搜索 + 热度排序 | 纯列表浏览，缺乏个性化引导 |
| AI 对话 | 小美顾问 + 运营助手双 Agent | 上下文孤立，不感知用户当前浏览状态 |
| 运营看板 | 营收/流量/评价/退款等 9 大维度 | 仅面向运营端，用户端无数据反馈 |
| NL2SQL | 自然语言查数据库 | 仅运营可用，普通用户无感知 |
| 热门排行 | 多维度热度分排行 | 展示型功能，缺乏互动 |

---

## 二、功能改进建议（按优先级排列）

### 🔥 P0 — 立刻提升核心体验

#### 1. AI 智能匹配 · 肤色手型分析

**痛点**：用户面对几十款美甲无从下手，试戴效率低，试了不喜欢就流失。

**方案**：利用 MediaPipe 手部关键点数据，自动分析用户手部特征（肤色冷暖调、手指长度比例、甲床形状），匹配数据库中最佳款式。

```
用户上传手图 → MediaPipe 提取 21 个关键点 →
  ① 肤色分析：提取手指皮肤区域 RGB 均值 → 映射冷暖色调 → 推荐适配色系
  ② 手型分析：手指长度比例 → 判断修长/圆润型 → 推荐适配甲型（方圆/尖圆/椭圆）
  ③ 甲床分析：甲床宽高比 → 推荐图案密度（简约/繁复）

输出：匹配度评分 + 推荐理由卡片
```

**技术可行性**：MediaPipe 已在项目中集成，只需增加关键点分析逻辑和配色映射表。

**竞赛亮点**：AI + CV 深度融合，有学术展示价值。

---

#### 2. 款式对比模式

**痛点**：用户一次只能试戴一款，无法直观对比效果，反复切换体验差。

**方案**：在试戴页增加"对比模式"，支持 2~4 款并排展示。

```
交互流程：
  ✅ 用户选择 2-4 款加入对比列表
  ✅ 一键生成全部试戴结果
  ✅ 并排展示（自适应网格布局）
  ✅ 支持投票打分 / 标注优缺点
  ✅ AI 自动生成对比总结
```

**技术改动**：前端新增 CompareView 组件，后端 try-on 接口支持批量。

---

#### 3. 上下文感知的 AI 对话

**痛点**：当前 AI 对话完全不感知用户在前端浏览/试戴了什么，回答泛化。

**方案**：在对话请求中注入当前页面上下文。

```json
// 请求扩展
{
  "message": "这款适合约会吗？",
  "context": {
    "current_style": { "id": 5, "name": "星空渐变", "category": "渐变" },
    "current_page": "tryon",
    "recent_tryons": ["星空渐变", "经典红"]
  }
}
```

**效果**：AI 能基于用户正在看的款式给出针对性建议，如 "星空渐变搭配裸色打底会更显白，你要试试吗？"

---

### 📌 P1 — 构建用户粘性

#### 4. 美甲日记 · Style Journey

**痛点**：试戴完就走，没有记录，没有回忆，没有复访动机。

**方案**：自动记录每次试戴 → 生成可视化的"美甲日记"时间线。

```
功能设计：
  📅 时间线视图：按日期分组展示试戴历史
  🏷 自动标签：#试过 #喜欢 #已购买（AI 根据行为推断）
  📊 风格画像：雷达图展示用户偏好（颜色/风格/形状偏好分布）
  🔄 回忆推荐：AI 根据历史偏好推荐"你可能会喜欢"
  📤 分享卡片：将试戴结果生成精美分享图
```

**留存钩子**：每周推送"本周热门新款 → 点击试戴"，形成使用习惯。

---

#### 5. 场景魔法师 · Occasion Wizard

**痛点**：用户按分类浏览需要自己做决策，决策成本高。

**方案**：用户告诉 AI 场景（约会/通勤/派对/婚礼/度假/面试），AI 从款式库中智能匹配并解释推荐理由。

```
交互：
  "明天要去参加闺蜜婚礼，穿香槟色礼服，推荐一款美甲"
  →
  AI 分析：正式场合 + 香槟色搭配 + 用户历史偏好
  →
  推荐 TOP 3 + 理由卡片（配色原理 + 场合适配度 + 用户偏好匹配度）
```

**差异化**：不同于简单关键词匹配，结合色彩理论、场合礼仪、用户画像的多维度推荐。

---

#### 6. 社区分享与互动

**痛点**：单机应用，没有社交传播路径，无法病毒式增长。

**方案**：轻量社交功能，用户分享试戴结果，其他用户可点赞/评论/"我也试试"。

```
核心功能：
  📸 试戴结果一键生成分享卡片（精美排版）
  ❤️ 点赞 + 收藏 + 试同款（点击直接跳转试戴）
  🏆 周榜/月榜：最受欢迎试戴作品
  👥 关注机制：关注风格相近的用户
```

**传播设计**：分享卡片带二维码，扫码进入 H5 试戴体验页（无需下载）。

---

### 💡 P2 — 差异化创新

#### 7. AI 美甲设计师 · Text-to-Nail

**痛点**：款式库是固定的，用户可能有"就想画一朵小雏菊"这样的具体需求。

**方案**：利用 Bailian API 的文生图能力，根据文字描述生成美甲设计图，再合成到手上。

```
用户输入："淡紫色底色，上面画白色小雏菊，金色勾边"
  → Bailian 生成美甲图案
  → 自动分离手指区域（MediaPipe）
  → 合成试戴效果
```

**技术挑战**：需要 prompt engineering 确保生成的图案适合指甲形状。

**竞赛亮点**：AIGC + 个性化 = 展示前沿技术能力。

---

#### 8. 智能运营助手升级 · 从看数据到做决策

**痛点**：当前运营助手只能查数据+生成报告，缺乏"自动执行"能力。

**方案**：闭环运营 —— 数据洞察 → 策略生成 → 一键执行。

```
场景示例：
  运营："最近退款率上升了，帮我分析一下"
  AI：分析退款原因分布 + 关联款式 → 
      "「樱花粉」退款率 23% 高于平均，主要原因「颜色与图片不符」(占 68%)。
       建议：①暂停该款推荐位  ②更新实物图  ③联系供应商。
       是否执行建议①？"
  运营："执行"
  AI：调用 API → 更新推荐位配置 → 返回确认
```

**新增 Skills**：自动下架退款率超标款式、自动调价、自动生成营销文案。

---

#### 9. 实时 AR 试戴（WebRTC）

**痛点**：静态图片试戴不够直观，用户需要反复上传不同角度的手图。

**方案**：通过浏览器摄像头实时捕获手部画面，MediaPipe 实时追踪手部关键点，动态渲染美甲效果。

```
技术栈：
  WebRTC + Canvas → 实时视频流
  MediaPipe Hands (Web/JS 版本) → 21 点手部追踪
  WebGL → 实时渲染美甲图层（支持角度变换、光照）
```

**体验飞跃**：从"上传图片 → 等待 → 查看结果"到"伸出手 → 实时看到效果"。

## 四、预期效果

| 指标 | 当前 | 预期提升 |
|------|------|---------|
| 单次会话时长 | ~2 分钟 | ~8 分钟（对比模式+日记增加停留） |
| 试戴转化率 | 浏览→试戴 30% | 50%+（智能匹配降低决策门槛） |
| 复访率 | ~5% | 25%+（日记时间线 + 社区粘性） |
| AI 对话使用率 | 偶发 | 高频（上下文感知 + 场景推荐触发） |

## 五、对竞赛的意义

以上建议围绕三个竞赛评审维度设计：

- **技术创新**：MediaPipe 手部分析、AIGC 文生图、NL2SQL 闭环
- **产品完整性**：从浏览→试戴→决策→分享→复访的完整用户旅程
- **商业价值**：社区传播、个性化推荐、运营自动化，均可直接商业化
